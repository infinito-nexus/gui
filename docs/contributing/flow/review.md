# Review Requirements

You MUST open a Pull Request only when it is ready for real review.

The [Code](../code/tests/lint.md) page is the SPOT for repository-wide code quality expectations such as diff quality, maintainability, and conventions.

In addition to the PR checklist in [pull-request.md](pull-request.md), keep these repository-wide review rules in mind:

- You MUST keep commit history clear and reviewable.
- Auth, API boundaries, database migrations, and shared abstractions MUST receive extra review attention.
- You SHOULD attach screenshots, traces, logs, and migration notes when they materially help validate the change.
